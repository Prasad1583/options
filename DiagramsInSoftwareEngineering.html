Diagram Purpose

üß± 2. Design & Architecture Diagrams

Used to design how the system is structured internally.

Diagram Purpose
Class Diagram Models object-oriented classes and relationships
Component Diagram Shows modular parts (components) of a system
Sequence Diagram Shows time-based message interaction between components
Activity Diagram Represents workflows, decisions, and loops
State Machine Diagram Shows object states and transitions
Package Diagram Groups classes or components into packages
Deployment Diagram Shows hardware nodes and software components' deployment
Object Diagram Snapshot of objects and values at a specific point in time
Composite Structure Diagram Describes internal structure of a class or component
Communication Diagram Focuses on object interactions using message passing
Timing Diagram Shows object behavior over time (used in embedded/real-time systems)
