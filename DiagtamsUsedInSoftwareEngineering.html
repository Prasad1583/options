Software Engineering Diagrams Reference

A comprehensive guide to all diagrams used in software development

Requirement Analysis Diagrams

Use Case Diagram
Shows how users (actors) interact with the system and its functionality
Context Diagram
High-level view of the system and its external entities and interactions
User Story Map
Organizes user stories by goals and priorities in a visual layout
Domain Model Diagram
Conceptual view of entities, their attributes, and relationships

System Design & Modeling (UML)

Class Diagram
Defines classes, their attributes, methods, and relationships
Object Diagram
Snapshot of objects and their relationships at a specific moment
Component Diagram
Shows system components/modules and their interfaces
Package Diagram
Groups related classes/interfaces into packages and dependencies
Composite Structure
Describes internal structure of a class or component
Deployment Diagram
Maps software components to physical hardware nodes

Behavioral Diagrams

Activity Diagram
Models workflows, business processes with decisions and loops
State Machine
Shows states of an object and transitions between them
Sequence Diagram
Illustrates object interactions in chronological sequence
Communication
Focuses on message passing between objects

Architecture & Infrastructure

Logical Architecture
Abstract view of system modules and components
Physical Architecture
Actual hardware, network layout and deployment
Microservices
Service boundaries, APIs, and database relationships
Serverless
Cloud components and event-driven functions architecture

Testing & Quality

Test Case Diagram
Visual representation of test cases and their flows
Decision Table
Logical combinations of inputs and expected outputs
Boundary Value
Tests focused on input boundaries and edge cases

Data Modeling

Entity-Relationship
Defines data schema, entities, attributes and relationships
Data Flow Diagram
Shows how data moves between processes in a system
Database Schema
Physical structure of database tables and constraints

DevOps & CI/CD

CI/CD Pipeline
Visualizes workflow from code commit to production deployment
Infrastructure as Code
Shows cloud resource provisioning and configuration
Version Control
Illustrates branching, merging and release strategies

Security & Compliance

Threat Modeling
Identifies and mitigates security vulnerabilities (STRIDE)
Access Control
Maps user roles to resource permissions
Security Architecture
Shows trust zones, firewalls, and encryption points

Business Process & Workflow

BPMN
Standard notation for business process modeling
Swimlane
Allocates workflow tasks to roles or departments
Workflow
Shows steps, decisions and flow of tasks

Project Management & Agile

Gantt Chart
Timeline view of tasks, dependencies and milestones
Kanban Board
Visualizes task progression (To-do, In Progress, Done)
Burndown Chart
Tracks remaining work in a sprint or project
Product Roadmap
High-level plan of features and milestones
